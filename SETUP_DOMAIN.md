# Настройка домена avito.site и SSL сертификата

## Шаг 1: Настройка домена в панели управления Beget

1. Войдите в панель управления Beget: https://cp.beget.com

2. Перейдите в раздел **"Домены и поддомены"**

3. Добавьте домен **avito.site**:
   - Нажмите "Добавить домен"
   - Введите: `avito.site`
   - Выберите каталог: `~/avito.site` или `~/public_html/avito.site`
   - Сохраните настройки

4. Настройте DNS записи (если домен не на Beget):
   - A-запись: `@` → IP адрес сервера Beget
   - A-запись: `www` → IP адрес сервера Beget
   - Или используйте NS записи Beget для делегирования домена

## Шаг 2: Установка SSL сертификата

### Вариант А: Бесплатный SSL через Let's Encrypt (рекомендуется)

1. В панели управления Beget перейдите в **"SSL сертификаты"**

2. Выберите домен **avito.site**

3. Нажмите **"Установить Let's Encrypt"** или **"Получить бесплатный SSL"**

4. Дождитесь установки сертификата (обычно 1-5 минут)

5. Включите **"Принудительное перенаправление на HTTPS"** (опционально, но рекомендуется)

### Вариант Б: Через командную строку (альтернатива)

Если в панели нет опции, можно установить через SSH:

```bash
# Подключитесь к серверу
ssh username@avito.site.beget.tech

# Подключитесь к Docker
ssh localhost -p222

# Установите certbot (если доступен)
# Обычно SSL настраивается через панель управления
```

## Шаг 3: Настройка проекта для домена

### 3.1. Обновите пути в проекте

Убедитесь, что проект находится в правильном каталоге:
```bash
cd ~/avito.site
# или
cd ~/public_html/avito.site
```

### 3.2. Обновите .htaccess

Откройте `.htaccess` и убедитесь, что путь к Python правильный:
```apache
PassengerEnabled On
PassengerPython /usr/bin/python3

# Если используете venv:
# PassengerPython /home/u/username/avito.site/venv/bin/python3
```

### 3.3. Создайте симлинк public (если еще не создан)

```bash
ln -s public_html public
```

### 3.4. Настройте права доступа

```bash
chmod 755 ~/avito.site
chmod 644 ~/avito.site/*.py
chmod 755 ~/avito.site/tmp
```

## Шаг 4: Настройка приложения для работы с доменом

Приложение уже настроено для работы с любым доменом. При необходимости можно добавить проверку домена в `app.py`:

```python
# В app.py можно добавить (опционально):
ALLOWED_HOSTS = ['avito.site', 'www.avito.site']
```

Но это не обязательно - Flask работает с любым доменом по умолчанию.

## Шаг 5: Перезапуск приложения

После настройки домена и SSL перезапустите приложение:

```bash
touch tmp/restart.txt
```

Или через панель управления Beget в разделе "Управление сайтами".

## Шаг 6: Проверка работы

1. Откройте в браузере: **https://avito.site**
2. Проверьте, что SSL сертификат активен (замочек в адресной строке)
3. Убедитесь, что сайт открывается корректно

## Решение проблем

### Домен не открывается
- Проверьте DNS записи (может потребоваться до 24 часов для распространения)
- Убедитесь, что домен добавлен в панели управления
- Проверьте, что каталог сайта указан правильно

### SSL сертификат не работает
- Убедитесь, что DNS записи настроены правильно
- Проверьте, что домен указывает на сервер Beget
- Подождите несколько минут после установки сертификата
- Попробуйте переустановить сертификат через панель

### Ошибка 500 Internal Server Error
- Проверьте логи ошибок в панели управления
- Убедитесь, что все зависимости установлены
- Проверьте права доступа к файлам
- Убедитесь, что `passenger_wsgi.py` настроен правильно

### Статические файлы не загружаются
- Проверьте, что создан симлинк `public -> public_html`
- Убедитесь, что каталог `static/` существует и доступен

## Дополнительные настройки

### Принудительное перенаправление на HTTPS

В `.htaccess` можно добавить (если не включено в панели):
```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

### Настройка www

Если нужно перенаправлять www на основной домен или наоборот, добавьте в `.htaccess`:
```apache
# Перенаправление www на основной домен
RewriteCond %{HTTP_HOST} ^www\.avito\.site$ [NC]
RewriteRule ^(.*)$ https://avito.site/$1 [R=301,L]
```
