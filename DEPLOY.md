# Инструкция по развертыванию на хостинге Beget

## Шаг 1: Загрузка проекта

1. Загрузите все файлы проекта в каталог вашего сайта через FTP или SSH
2. Рекомендуемый путь: `~/yourdomain.com/` или `~/public_html/`

## Шаг 2: Подключение к Docker окружению

Подключитесь к серверу по SSH:
```bash
ssh username@yourdomain.beget.tech
```

Затем подключитесь к Docker-контейнеру:
```bash
ssh localhost -p222
```

## Шаг 3: Установка зависимостей

Перейдите в каталог проекта:
```bash
cd ~/yourdomain.com
```

### Вариант А: Использование виртуального окружения (рекомендуется)

Создайте виртуальное окружение:
```bash
python3 -m venv venv
```

Активируйте его:
```bash
source venv/bin/activate
```

Установите зависимости:
```bash
pip install -r requirements.txt --ignore-installed
```

### Вариант Б: Локальная установка (как в инструкции Beget)

Если виртуальное окружение не используется, установите зависимости с флагом `--user`:
```bash
pip3 install -r requirements.txt --user --ignore-installed
```

**Важно:** Флаг `--user` является обязательным при установке модулей на хостинге Beget.

Пример установки Flask:
```bash
pip3 install flask --user --ignore-installed
```

После установки пути автоматически определяются в `passenger_wsgi.py`. 
Если нужно указать путь вручную, используйте формат:
```python
sys.path.append('/home/u/username/.local/lib/python3.X/site-packages')
```
Где `u` - первая буква вашего логина, `username` - ваш логин, `X` - версия Python (например, 3.11).

## Шаг 4: Настройка файлов для хостинга

### 4.1. Создание симлинка public

Для корректной отдачи статического контента средствами Nginx создайте симлинк:
```bash
ln -s public_html public
```

**Важно:** Если симлинк уже существует, команда выдаст ошибку. Это нормально - значит симлинк уже создан.

### 4.2. Настройка .htaccess

Откройте файл `.htaccess` и укажите правильный путь к Python:

**Если используете виртуальное окружение:**
```apache
PassengerEnabled On
PassengerPython /home/u/username/yourdomain.com/venv/bin/python3
```

**Если используете системный Python:**
```apache
PassengerEnabled On
PassengerPython /usr/bin/python3
```

**Если используете локально собранный Python:**
```apache
PassengerEnabled On
PassengerPython /home/u/username/.local/bin/python3
```

### 4.3. Настройка passenger_wsgi.py

Файл `passenger_wsgi.py` уже настроен и автоматически определяет пути. 

**Структура файла (как в инструкции Beget):**
```python
# -*- coding: utf-8 -*-
import sys, os

# Путь до проекта
sys.path.append('/home/u/username/yourdomain.com/')  # указываем директорию с проектом

# Путь до библиотек
sys.path.append('/home/u/username/.local/lib/python3.X/site-packages')  # путь до библиотек, куда поставили Flask

# Импорт приложения
from app import app as application  # когда Flask стартует, он ищет application
```

**В нашем проекте:**
- Пути определяются автоматически
- Поддерживается как виртуальное окружение, так и локальная установка
- SocketIO автоматически отключается для хостинга
- База данных инициализируется автоматически

Если нужно указать пути вручную, отредактируйте `passenger_wsgi.py` согласно примеру выше.

### 4.4. Создание каталога tmp

Каталог `tmp` уже создан в проекте. Если его нет:
```bash
mkdir tmp
touch tmp/restart.txt
```

## Шаг 5: Настройка прав доступа

Для работы приложения необходимо настроить права доступа через Файловый менеджер в панели управления Beget:

1. Войдите в панель управления
2. Откройте Файловый менеджер
3. Настройте общий доступ к каталогам `.local` (если используется локальная установка)

## Шаг 6: Настройка базы данных

База данных SQLite будет создана автоматически при первом запуске в каталоге `instance/`.

Если нужно использовать другую БД, измените настройки в `app.py`:
```python
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///avita.db'
```

## Шаг 7: Перезапуск приложения

После всех изменений в корневом каталоге сайта выполните команду:
```bash
touch tmp/restart.txt
```

Это создаст/обновит файл перезапуска, что заставит Passenger перезагрузить приложение.

**Альтернативно:** Перезапуск можно выполнить через панель управления Beget в разделе "Управление сайтами".

## Решение проблем

### Ошибка "No module named 'flask'"
- Проверьте путь к Python в `.htaccess`
- Убедитесь, что зависимости установлены в правильное окружение
- Проверьте пути в `passenger_wsgi.py`

### Ошибка "Permission denied"
- Настройте права доступа через Файловый менеджер
- Проверьте права на каталоги: `chmod 755` для каталогов, `chmod 644` для файлов

### Статические файлы не загружаются
- Убедитесь, что создан симлинк `public -> public_html`
- Проверьте настройки Nginx в панели управления

### База данных не создается
- Проверьте права на запись в каталог `instance/`
- Убедитесь, что каталог существует: `mkdir -p instance`

### Ошибка "PermissionError: Operation not permitted" при запуске
- **Эта ошибка исправлена!** SocketIO теперь автоматически отключается на хостинге
- Приложение работает через WSGI без прямого запуска сервера
- Чат работает через HTTP polling вместо WebSocket

### SocketIO и WebSocket
- На хостинге Beget SocketIO отключен по умолчанию (через `passenger_wsgi.py`)
- Чат автоматически переключается на polling (проверка новых сообщений каждые 2 секунды)
- Это нормальное поведение и не влияет на функциональность

## Дополнительные настройки

### Изменение SECRET_KEY

Для безопасности измените `SECRET_KEY` в `app.py`:
```python
app.config['SECRET_KEY'] = 'ваш-случайный-секретный-ключ'
```

### Настройка домена

В `app.py` можно добавить проверку домена (опционально):
```python
ALLOWED_HOSTS = ['yourdomain.com', 'www.yourdomain.com']
```

## Поддержка

При возникновении проблем обращайтесь в поддержку Beget через тикет из панели управления.
